¿Como construirías un sitio estático?

¿Con HTML estático? ... ¡Mala idea!

Beneficios Contras
  • Mayor Seguridad
  • Mayor Velocidad
  • Desarrollo inicial sencillo
  • Mantención lenta
  • Las requerimientos siempre estan cambiando.
  • Por lo anterior, los sitios también lo hacen.
  • HTML no tiene una forma de abstraer estructuras repetitivas.
  • ... O incluir trozos de HTML

¿Con algún lenguaje dinámico?
(PHP, Ruby, Node, Etc.) ... ¡Algo mejor!

Beneficios Contras
  • Permite abstraer las estructuras repetitivas.
  • Realizar cambios es mucho mas fácil.
  • El servidor debe volver a procesar el sitio cada vez que es servido aún cuando este no cambia.
  • Más expuesto a sufrir de alguna vulnerabilidad de seguridad

¿Y si pudiéramos obtener lo mejor de ambos mundos?
